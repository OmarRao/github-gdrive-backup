Repository Secrets for GitHub Actions

Add these 5 secrets under Settings → Secrets → Actions to enable the scheduled and manual backup workflows.

Secret Value / Where to get it Status
GH_BACKUP_TOKEN GitHub PAT with repo, workflow, read:org, read:user unknown
GH_USER Your GitHub username or org to back up unknown
GDRIVE_FOLDER_ID Google Drive folder ID (from Drive URL) unknown
GOOGLE_CLIENT_SECRET Full JSON content of credentials/google-client-secret.json unknown
GOOGLE_TOKEN Full JSON content of credentials/google-token.json (from one-time CLI auth) unknown
Open Secrets Settings →
Quick Setup Guide
  1. Google Cloud OAuth app — go to Google Cloud Console, create an OAuth 2.0 Client ID (Web application), add https://omarrao.github.io as an authorised JavaScript origin and http://localhost as an authorised redirect URI.
  2. One-time CLI auth — clone the repo locally, run python get_token.py, follow the browser prompt, then copy the contents of credentials/google-token.json into the GOOGLE_TOKEN Actions secret.
  3. Add all 5 secrets — see the Actions Secrets tab above for the full list.
  4. Trigger a backup — go to the Backup tab, select your repos, and click Trigger Backup Workflow. The workflow runs automatically every day at 02:00 UTC.
  5. Connect Drive in the dashboard — paste your Google OAuth Client ID in the Google Drive settings tab and click Sign in with Google to browse backup sessions live.
